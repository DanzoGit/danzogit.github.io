Works with version 2.0 and above, and with versions 1.0–1.4.

For game versions 1.0–1.4, requires mod version 1.3.2 or earlier.

The mod plays sound alerts:

1) When a screamer spawns.
2) When a screamer spawns a horde of zombies.
3) When a wandering horde spawns.

There are 57 sounds to choose from.
You can listen to them and select the appropriate one.
They are located in the Sounds folder.
All sounds are listed in the sounds.txt file.

The configuration is contained in the config.json file.

Sound volume:
Values range from 0 to 1. If the value is 0, the sound will not be played.
SpawnScoutsVolume - screamer spawn.
SpawnHordeNearVolume - screamer spawns a horde of zombies.
WanderingHordeVolume - wandering horde spawn.

Types of alerts:
You can replace the sounds with any listed in sounds.txt.
SpawnScoutsSound - screamer spawn.
SpawnHordeNearSound - screamer spawns a horde of zombies.
WanderingHordeSound - wandering horde spawn.

For each alert type, you can set any number of sounds to be played in order.
An empty value of type "" is ignored.
If you set RandomizeSounds to true, the mod will try to randomize the sounds of alerts (repetitions are possible).

Alert sounds can be made to play within a certain radius on the map:
Several parameters in the configuration are responsible for this behavior.

SoundAttachedToPosition: the default value for it is auto - in a single-player game, the sound will spread across the entire map, in multiplayer - only at the spawn location of the screamer, the wandering horde, and the zombie spawn point from the screamer.
If set to disabled, the sound will always spread across the entire map, regardless of the number of players.
If set to enabled, the sound will always be attached to the position.

SoundRadius: the default value is 120. This is the radius within which the alert sound will spread.
Keep in mind that a screamer spawns at a distance of 80 meters from the player, and a wandering horde - 92 meters.
If you set the radius too small, you won't hear the sound at all.
The optimal value is 100-120.

DebugRadius: when toggled to true, it displays a large red sphere that visualizes the sound radius. Inside this sphere is another small yellow sphere that indicates the center of the large sphere.

When playing a sound within a certain radius, the player's elevation (height) is not taken into account. This is done intentionally, because if the player is too low (for example, in a mine) or too high (on the roof of a skyscraper or on a flying vehicle), the player would not hear the alert sound.
Also, the player's movement during the playback of the alert sound is not taken into account. That is, the sound will not be interrupted if the player goes outside the sound radius. This is also done intentionally.

P.S. Initially, I wanted to implement spatial sound, but during testing, I had to abandon it. If the player moves, the sound doesn't behave very well, and additionally, the volume can be too weak to hear the alert.

There is an option to add a custom asset bundle.
It needs to be placed in the Resources folder.
The default name for it is CustomAlertSounds.unity3d.
This can be changed in the config file.
You can also enable its loading there.

For creating the asset bundle, I used:
Unity (Editor version: 2022.3.38f1)
Unity3D Resource Bundle Multi-Asset Exporter
The camera and sound source are not needed yet; these settings are handled in the mod's code.

﻿Server settings:
The parameter UseServerConfig in the config is responsible for this. By default, it is set to false (disabled).
Switching it to true will force clients to use the server's settings.
For custom alerts to work for clients, they need to download the sound pack and place it in the appropriate folder.
No config changes are required on the client side.
If the custom pack is not found in the Resources folder, the default sounds will be used, but the server's volume settings will be applied.
When clients disconnect from the server, their settings specified in their config will be restored.
Changing the UseServerConfig parameter does not affect the client side.

Alerts should work in multiplayer.
The mod needs to be installed for both clients and the server.
The mod is incompatible with earlier or later versions of the mod.
The same version must be installed on both clients and the server.
Alert settings and their volume can be individually configured for each client (unless the server enforces the use of server settings).
I only tested it on a dedicated server where I was the only player.

For the mod to function, the 0_TFP_Harmony folder is required in the Mods folder.
Requires disabling EAC.
